# Plan 02-04: Refatoração FloatingChat — Histórico Backend + Context Awareness

## Goal

O FloatingChat atualmente persiste mensagens em localStorage (TTL 24h, máximo 20 mensagens) e não tem consciência da página em que o usuário está. Esta refatoração conecta o chat à Sessions API criada na Phase 01 (`POST/GET/PATCH /api/llm/sessions`) e injeta automaticamente o contexto da portaria aberta no system prompt quando o usuário está em `/portarias/:id`.

## Context

**State atual do FloatingChat:**
- Histórico: `localStorage` com chave `docs_chat_messages_{userId}`, TTL 24h, máx 20 mensagens
- Contexto: somente `userAuth` (`nome`, `email`, `role`) — sem contexto de página
- Sessions API: não usada — criada na Phase 01 mas o FloatingChat nunca foi conectado
- Localização: `apps/web/src/components/chat/FloatingChat.tsx`

**Sessions API (Phase 01 — pronto para uso):**
- `POST /api/llm/sessions` — cria sessão vazia com `{ provider?, model?, temperature?, maxTokens? }`
- `GET /api/llm/sessions/:id` — retorna sessão com mensagens
- `PATCH /api/llm/sessions/:id` — salva mensagens e atualiza stats
- `DELETE /api/llm/sessions/:id` — deleta sessão
- Autenticação: JWT cookie (já presente no FloatingChat via `api` axios instance)

**Context awareness:** Ao abrir o chat em `/portarias/abc123`, o system prompt deve incluir o estado atual da portaria (título, status, número). A rota `GET /api/portarias/:id` já existe e retorna esses campos.

**Sobreposição de campos:** Após análise do código, não existe sobreposição visual no CSS do FloatingChat. O bug reportado provavelmente se referia a uma versão anterior. A refatoração mantém o layout atual (stack vertical: seletor → preview DOCX → textarea → botão enviar).

**Limitações desta refatoração:**
- Histórico server-side NÃO substitui o DOCX anexado — esse fluxo (upload + análise) é mantido intacto
- Sessão nova a cada abertura do chat (reutilizar seria complexo e desnecessário para v1)
- Ações expandidas (criar portaria, navegar) ficam para fase futura — complexidade desproporcional ao ganho

## Tasks

### Task 1: Criar hook useChatSession para gerenciar lifecycle da sessão

**File:** `apps/web/src/hooks/use-chat-session.ts` (novo)

**What:** Criar um hook React que encapsula a criação, carregamento e salvamento da sessão de chat via Sessions API. Isola toda a lógica de persistência do componente FloatingChat.

**Why:** O FloatingChat já tem ~750 linhas. Extrair a lógica de sessão para um hook mantém o componente legível e torna a lógica de sessão testável isoladamente.

**How:**
1. Criar `apps/web/src/hooks/use-chat-session.ts` com:

```typescript
import { useState, useCallback } from 'react'
import api from '@/lib/api'

interface SessionMessage {
    id: string
    role: 'user' | 'assistant'
    content: string
    provider?: string
    model?: string
    tokens?: number
    error?: boolean
}

interface UseChatSession {
    sessionId: string | null
    isCreating: boolean
    createSession: (provider?: string, model?: string) => Promise<string | null>
    saveMessages: (sessionId: string, messages: SessionMessage[]) => Promise<void>
}

export function useChatSession(): UseChatSession {
    const [sessionId, setSessionId] = useState<string | null>(null)
    const [isCreating, setIsCreating] = useState(false)

    const createSession = useCallback(async (provider?: string, model?: string): Promise<string | null> => {
        setIsCreating(true)
        try {
            const res = await api.post('/api/llm/sessions', {
                titulo: `Chat ${new Date().toLocaleDateString('pt-BR')}`,
                ...(provider && { provider }),
                ...(model && { model }),
            })
            if (res.data?.success && res.data?.data?.id) {
                setSessionId(res.data.data.id)
                return res.data.data.id
            }
            return null
        } catch (err) {
            console.error('[useChatSession] Falha ao criar sessão:', err)
            return null
        } finally {
            setIsCreating(false)
        }
    }, [])

    const saveMessages = useCallback(async (sid: string, messages: SessionMessage[]): Promise<void> => {
        try {
            // Salva apenas as últimas 50 mensagens para evitar payload grande
            const toSave = messages.slice(-50)
            await api.patch(`/api/llm/sessions/${sid}`, {
                messages: toSave.map(m => ({
                    role: m.role,
                    content: m.content,
                    metadata: {
                        provider: m.provider,
                        model: m.model,
                        tokens: m.tokens,
                        error: m.error,
                    }
                }))
            })
        } catch (err) {
            // Falha silenciosa — não interrompe o chat
            console.warn('[useChatSession] Falha ao salvar mensagens:', err)
        }
    }, [])

    return { sessionId, isCreating, createSession, saveMessages }
}
```

### Task 2: Criar hook usePageContext para extrair contexto da portaria atual

**File:** `apps/web/src/hooks/use-page-context.ts` (novo)

**What:** Hook que lê `window.location.pathname`, detecta se o usuário está em `/portarias/:id`, busca os dados da portaria via API e retorna uma string formatada para injeção no system prompt.

**Why:** O context awareness é o principal valor agregado desta refatoração. Sem um hook dedicado, a lógica de detecção de rota e fetch da portaria poluiria o FloatingChat.

**How:**
1. Criar `apps/web/src/hooks/use-page-context.ts` com:

```typescript
import { useState, useEffect } from 'react'
import { useLocation } from '@tanstack/react-router'
import api from '@/lib/api'

interface PageContext {
    systemContext: string | null
    isLoading: boolean
}

export function usePageContext(): PageContext {
    const location = useLocation()
    const [systemContext, setSystemContext] = useState<string | null>(null)
    const [isLoading, setIsLoading] = useState(false)

    useEffect(() => {
        // Detectar rota de portaria: /portarias/:id ou /portarias/revisao/:id
        const portariaMatch = location.pathname.match(/\/portarias\/(?:revisao\/)?([a-z0-9]+)/)
        
        if (!portariaMatch) {
            setSystemContext(null)
            return
        }

        const portariaId = portariaMatch[1]
        setIsLoading(true)

        api.get(`/api/portarias/${portariaId}`)
            .then((res) => {
                const p = res.data?.data
                if (!p) return

                const ctx = [
                    `[CONTEXTO DA PÁGINA ATUAL]`,
                    `O usuário está visualizando a portaria: "${p.titulo}"`,
                    `ID: ${p.id}`,
                    p.numeroOficial ? `Número oficial: ${p.numeroOficial}` : 'Número oficial: ainda não publicada',
                    `Status: ${p.status}`,
                    p.modelo?.tipoDocumento ? `Tipo: ${p.modelo.tipoDocumento}` : '',
                    p.secretaria?.nome ? `Secretaria: ${p.secretaria.nome}` : '',
                    ``,
                    `Se o usuário perguntar sobre "esta portaria", "este documento" ou "este arquivo", refira-se aos dados acima.`,
                ].filter(Boolean).join('\n')

                setSystemContext(ctx)
            })
            .catch(() => {
                // Sem contexto se a portaria não for acessível — não bloqueia o chat
                setSystemContext(null)
            })
            .finally(() => setIsLoading(false))
    }, [location.pathname])

    return { systemContext, isLoading }
}
```

### Task 4: Adicionar ações básicas de contexto (botões de ação rápida)

**File:** `apps/web/src/components/chat/FloatingChat.tsx`

**What:** Quando o usuário está em uma rota de portaria (`/portarias/:id`), exibir abaixo do input um conjunto de botões de ação rápida:
- **"Ver esta portaria"** — navega para `/_sistema/administrativo/portarias/:id`
- **"Nova portaria"** — navega para `/_sistema/administrativo/portarias/novo`

**Why:** O ROADMAP pede "ações expandidas". Ações básicas de navegação contextual têm alto valor e baixa complexidade — o `portariaId` já está disponível via `usePageContext`.

**How:**
1. Exportar `portariaId` (além de `systemContext`) do hook `usePageContext`:
   ```typescript

       systemContext: string | null

   // No useEffect, após detectar o match:
   ```

2. No `FloatingChat.tsx`, importar `useNavigate` do `@tanstack/react-router`:
   ```typescript

   ```
   E dentro do componente:

3. Adicionar o bloco de ações rápidas no JSX, logo acima do `<textarea>` (na área de input do chat). Renderizar apenas quando `portariaId` não é nulo E o chat está aberto:
