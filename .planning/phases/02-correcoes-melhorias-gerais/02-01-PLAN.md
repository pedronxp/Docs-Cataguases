# Plan 02-01: PDF/Assinatura + Numeração

## Goal

Dois bugs críticos bloqueando o ciclo de vida dos documentos:

1. Após registrar uma assinatura, `pdfUrl` é zerado para `null` mas nenhuma regeneração é disparada — o próximo acesso ao PDF falha com "ficheiro não disponível".
2. Documentos publicados podem aparecer como "001/2026" em vez de "PORT-001/2026" dependendo do valor de `formato_base` no banco.

## Context

**Bug PDF (BUG-01):** `apps/api/src/app/api/portarias/[id]/assinar/route.ts` linhas 134-135 zeram `pdfUrl: null` e `docxRascunhoUrl: null`. A intenção era forçar regeneração com carimbo de assinatura, mas nenhuma chamada de regeneração existe. O `/publicar` já regenera o PDF com número oficial — logo o PDF intermediário não precisa ser invalidado nesse passo. Solução: remover essas duas linhas da transação.

**Bug Numeração (BUG-02):** `apps/api/src/services/numeracao.service.ts` usa `livro.formato_base` como template. Livros criados manualmente no banco com `formato_base = '{N}/{ANO}'` (sem prefixo `PORT-`) geram "001/2026". O padrão correto definido no código (linha 21) é `'PORT-{N}/{ANO}'`. Solução: normalizar em memória antes de aplicar o template, sem tocar no banco, e adicionar uma migration de correção para livros legados.

## Tasks

### Task 1: Remover zeragem de pdfUrl no assinar/route.ts

**File:** `apps/api/src/app/api/portarias/[id]/assinar/route.ts`

**What:** Remover as linhas 134-135 que zeram `pdfUrl: null` e `docxRascunhoUrl: null` do objeto `data` do `prisma.portaria.update`.

**Why:** O PDF existente (gerado na submissão) continua válido até a publicação. O `/publicar` já regenera o PDF com o número oficial e carimbo correto. Zerar aqui causa "ficheiro não disponível" sem nenhum benefício.

**How:**
1. Abrir `apps/api/src/app/api/portarias/[id]/assinar/route.ts`
2. Localizar o bloco `data: { ... }` dentro do `tx.portaria.update` (linhas ~127-137)
3. Remover apenas as duas linhas:

4. O objeto `data` resultante deve ter apenas: `assinaturaStatus`, `assinaturaJustificativa`, `assinaturaComprovanteUrl`, `status: 'PRONTO_PUBLICACAO'`, `assinadoPorId`, `assinadoEm`
5. Verificar que o comentário `// Clear cached pdf so it can be regenerated with signature` também é removido (estava junto com `pdfUrl: null`)

### Task 2: Normalizar formato_base no NumeracaoService

**File:** `apps/api/src/services/numeracao.service.ts`

**What:** Após carregar o livro da query SQL (linha ~44), normalizar `livro.formato_base` em memória para garantir prefixo `PORT-` se ausente.

**Why:** Livros criados manualmente ou via seed antigo podem ter `formato_base = '{N}/{ANO}'`. A normalização em memória (sem alterar o banco) protege todos os fluxos sem exigir migration obrigatória imediata.

**How:**
1. Localizar a linha `let livro = livros[0]` (aprox. linha 44 do `numeracao.service.ts`)
2. Logo após essa linha, adicionar a normalização:
   ```typescript
   // Normaliza formato legado sem prefixo (ex: '{N}/{ANO}' → 'PORT-{N}/{ANO}')
   if (livro.formato_base && !livro.formato_base.startsWith('PORT-')) {
       livro = { ...livro, formato_base: `PORT-${livro.formato_base}` }
   }
   ```
3. Garantir que `livro` é reatribuível (já é `let`, não `const`)
